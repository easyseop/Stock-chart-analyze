# 화면 설계 명세서 (Claude Design 요청용) — v1

> 차트만으로 군중심리를 읽어 **리스크 관리 관점**의 매매 신호를 주는 스캐너의 웹 화면 명세.
> 목적: 디자이너(Claude Design)가 이 문서만으로 화면을 만들 수 있도록 **모든 화면·요소·상호작용·상태**를 세세히 기술.
> 현재 구현은 정적 HTML(plotly) + GitHub Pages 배포. 데이터는 매일 자동 갱신.

---

## 0. 제품 한 줄 / 디자인 원칙
- **한 줄**: "차트 신호로 종목을 랭킹하고(스크리너), 클릭하면 상세 차트로 근거를 본다. 예측기가 아니라 손절·비중 중심의 리스크 통제 도구."
- **디자인 원칙**
  1. **정직성 우선**: 과장 금지. "매수하라"가 아니라 "이런 상태다 + 손절은 여기"를 보여준다.
  2. **정보 밀도 + 스캔 용이**: 수백 종목을 빠르게 훑고 정렬/필터로 솎아낸다.
  3. **색은 의미로만**: 초록=강세/전환후보, 빨강=약세/회피, 회색=중립/관망. 그 외 장식색 최소.
  4. **모바일에서도 열람**: 폰으로 출퇴근길에 본다. 표는 가로 스크롤, 상세는 세로 스택.
  5. **다크 헤더 + 라이트 본문**: 헤더/카드는 짙은 남색(#0f172a), 본문 배경 #f1f5f9.

## 화면 1. 스크리너(메인/목록) — `index.html`
> 전 종목(현재 목표 200, 확장 가능)을 한 표로. **가장 많이 쓰는 화면.**

### 레이아웃 (위→아래)
1. **헤더(다크)**: 제목 "종목 스크리너" + 부제 "차트 신호 랭킹". 보조줄: "{N}종목 · 헤더 클릭=정렬 · 칩=필터 · 종목명 클릭=상세". 우측에 마지막 갱신 시각(데이터 기준일) 배지.
2. **필터 바(칩)**: `전체` `🟢전환 {n}` `⚪관망 {n}` `🔴회피 {n}` (+ 향후 `🟢매수권` `신고가권` `시장상승만`). 선택 칩은 채워진 파란 배경. 다중선택은 v2.
3. **검색창**(v1.1): 종목명/코드 즉시 필터.
4. **표(메인)**: 아래 컬럼. 헤더 sticky(스크롤해도 고정). 행 hover 하이라이트.

### 표 컬럼 (각 컬럼: 의미 · 포맷 · 정렬)
| # | 컬럼 | 의미 | 포맷/예 | 정렬 | 색 |
|---|---|---|---|---|---|
| 1 | 신호 | 종합 게이지 | 🟢강세 / 🟢 / ⚪중립 / 🔴 / 🔴공포 | 텍스트 | — |
| 2 | 종목 | 이름+코드(상세 링크) | 엔페이즈 ENPH | 가나다 | 링크 파랑 |
| 3 | 점수 | 정규화 −100~+100 | +33 / −14 | **숫자(기본 내림차순)** | 전환=초록·회피=빨강 |
| 4 | 시장 | 시장(지수) 방향 | 상승 / 하락 | 텍스트 | — |
| 5 | RS | 지수 대비 상대강도(60일) | +10% / −51% | 숫자 | +초록 −빨강 |
| 6 | 신고가 | 52주 고가 대비 | −5% / −44% | 숫자 | — |
| 7 | 판정 | 한 줄 결론 | "하락추세선 임박 — 돌파 확인 시 전환 매수 후보" | — | 회색 |

- **행 색 규칙**: 좌측 4px 컬러바 — 전환후보=초록, 회피=빨강, 관망=회색. 점수 셀은 부호색.
- **정렬**: 헤더 클릭 → 해당 컬럼 정렬(숫자 컬럼은 내림차순 우선, 다시 클릭 토글). 기본 = 점수 내림차순. 정렬 중인 헤더에 ▼/▲ 표시.
- **필터**: 칩 클릭 → 해당 버킷 행만 표시(나머지 display:none).
- **행 클릭/종목명 클릭** → 화면 2(상세)로 이동(`stocks/{code}.html`).

### 상태
- **로딩**: (정적이라 거의 없음) 표 스켈레톤 3~5줄.
- **빈 결과**(필터로 0건): "조건에 맞는 종목 없음" 안내.
- **데이터 없음**(수집 실패 종목): 회색 처리 + "데이터 없음" 배지(표에서 제외 가능).

### 모바일
- 표 가로 스크롤. 또는 **카드 리스트 전환**(한 종목=한 카드: 신호·이름·점수·RS·판정). 칩 필터는 가로 스크롤.

### 향후(디자인 반영 환영)
- 컬럼 커스터마이즈(보이기/숨기기), 즐겨찾기(★), 섹터 필터, 스파크라인(미니 가격선) 컬럼.

## 화면 2. 종목 상세 — `stocks/{code}.html`
> 스크리너에서 고른 종목 하나의 **차트 + 신호 근거**. 클릭 시에만 로드(무겁지 않게).

### 레이아웃 (위→아래)
1. **헤더(다크)**: `← 스크리너 목록`(뒤로) · 제목 "엔페이즈 ENPH · 🟢 정규화 +33" · 그 아래 한 줄 판정(verdict, 회색).
2. **지표 토글 바**: 라벨 "지표" + 토글 버튼들(아래 그룹). 켜진 건 파란 채움.
3. **타임프레임 바**: `일봉` `주봉` `월봉`(증권창식). 선택=짙은 채움. 우측 힌트 "추세는 프레임마다 다름".
4. **차트 영역**(흰 카드): 선택 타임프레임의 plotly 캔들 차트(아래 구성).
5. **상세 신호 카드**(접이식 `<details>`): 펼치면 텍스트 카드 전체(아래 항목).

### 2-A. 지표 토글 그룹 (각 버튼이 차트의 해당 레이어를 on/off)
| 키 | 라벨 | 포함 요소 | 기본 |
|---|---|---|---|
| ma | 이평선 | 타임프레임별 이동평균 3선 | ON |
| level | 핵심선 | 저항(박스상단)·핵심방어선·POC·52주고가 | ON |
| trade | 매매선 | 진입(기준가)·손절·목표(1:2) — 점선 | ON |
| trend | 추세선 | 하락/상승 추세선 + **미래 연장 점선** | ON |
| sr | 지지/저항 | 스윙 군집선(강도순 굵기) | OFF |
| fib | 피보나치 | 되돌림 0.236~0.786 | OFF |
| supply | 매물대 | 가로 거래량 분포 + 추정평단선 | OFF |

### 2-B. 차트 구성 (plotly, 2단: 가격 76% / 거래량 24%)
- **캔들**: 양봉 초록·음봉 빨강. 일봉은 **주말 빈칸 제거**(캔들 또렷).
- **이평선**(ma): 일=MA20/60/120, 주=MA10/20/60, 월=MA6/12/24.
- **박스권 음영**: 옅은 파랑 밴드(상단=저항, 하단=방어선).
- **밸류영역 음영**: 옅은 청록(VAL~VAH).
- **핵심선**(level): 저항/방어선(굵게)/POC(점)/52주고가(점) — 우측 끝에 라벨.
- **매매선**(trade): 진입(파랑 대시)·손절(빨강 대시 굵게)·목표(초록 대시).
- **추세선**(trend): 하락=빨강·상승=초록 longdash + **현재→미래 점선 연장**(추세 지속 시 선이 갈 자리).
- **지지/저항선**(sr): 점선, 강할수록 굵게.
- **피보선**(fib): 옅은 보라 점선 + 비율 라벨.
- **매물대**(supply): 좌측 가로 막대 히스토그램(거래량 분포) + 추정평단(점쇄선).
- **거래량 막대**: 양봉 초록·음봉 빨강.
- **호버**: `closest` — 마우스에 가장 가까운 **한 요소만** 표시. 지지/저항·매매선에 올리면 **그 선의 가격**과 용어 설명이 뜸. (정보 폭주 금지)
- **우측 여백**: 추세선 미래 연장 점선이 보이도록 약간 비워둠.

### 2-C. 상세 신호 카드(접이식) — 모든 줄 항목
헤더 줄: `[이름 코드]   심리: 게이지 (정규화 ±N점)`
- 국면: 추세장/횡보장/전환 (ADX 값) · 방향
- 추세: 정/역/혼조 + 다중TF(월·주·일 화살표) + 크로스 [점수]
- 시장방향: 상승장/하락장 [점수]
- 상대강도: 지수 대비 ±%/60일 [점수]
- 신고가: 52주고가 대비 ±% [점수]
- 모멘텀: RSI 값 [점수]
- 지지/저항: 박스 위치 [점수]
- 거래대금: 평균 대비 배수 [점수]
- 추세선: 상태(전환확정/돌파후횡보/임박/지속…) + 선 가격·거리 [점수]
- 위 저항 / 아래 지지 (가격·터치·강도)
- 매물대: VAL · POC · VAH
- 평단심리: 추정평단(미실현손익%) · 머리위물량% · 단기/장기평단
- 박스권: 하단~상단 (현재가, 상단대비 %)
- 핵심방어선: 가격 ⚠️ 이탈금지 (신뢰도·겹침)
- **판정 / 진입(기준) / 손절(ATR·방어선 중 택) / 1차 목표(손익비) / 비중(계좌 1% 리스크)**

> 디자인 포인트: 각 줄의 **[점수]는 +초록/−빨강/0회색 작은 배지**로. "판정/손절/목표"는 강조 박스로(가장 중요).

### 상태
- 데이터 부족(신규 상장): 일부 지표 "데이터 부족" 회색.
- 차트 로딩: plotly 스피너(기본).

### 모바일
- 토글 바·타임프레임 바 가로 스크롤. 차트 높이 축소. 카드는 가로 스크롤 가능한 `<pre>` 또는 줄바꿈 카드형.

## 화면 3. 백테스트 리포트 (선택/주간) — `backtest.html` 또는 상세의 탭
> "이 신호로 과거에 매매했으면?"의 결과. 매일 갱신 아님(무거움). 주간/수동.

### 구성
1. **요약 카드 3개**: 전체 / 🟢전환후보 / 일반매수 — 각 카드: **거래당 기대값(R, 대형 숫자)** + 거래수·승률·누적R·손익비(PF)·최대낙폭.
2. **누적손익 곡선**(R 단위): 전체·전환후보·일반매수 3선. 0선 점선. "우상향이어야 엣지" 캡션.
3. **종목별 표**: 종목·거래·승률·기대값·PF·최대낙폭.
4. **돌파 확인 필터 실험 표**: 전략별(무필터/거래량/RSI/콤보…) 기대값 비교. 양(+) 행은 초록 배경 강조.
5. **주의 문구**(필수, 정직성): "표본·생존편향 주의 · 슬리피지·수수료 미반영 · 승률보다 기대값·낙폭으로 해석".

> 디자인 포인트: 기대값(R)을 +면 초록 −면 빨강 대형 숫자로. 카드 상단 컬러바로 그룹 구분.

## 4. 공통 컴포넌트 / 상호작용
- **용어 툴팁**: 차트 요소 hover 시 용어 설명(glossary). 카드 하단 각주(접이식)로도 제공.
- **숫자 포맷터**: 통화별(USD 소수2 / KRW 정수). 퍼센트는 부호 포함(+/−).
- **게이지 칩**: 🟢강세/🟢/⚪중립/🔴/🔴공포 — 어디서나 동일 표기.
- **버킷 색**: 전환=초록 / 회피=빨강 / 관망=회색 — 스크리너·상세 공통.
- **빈/에러/로딩 상태**: 각 화면에 정의(위 참조).
- **접근성**: 색만으로 의미 전달 금지 → 텍스트 라벨 병기(예: "🔴 회피").

## 5. 데이터/갱신 모델 (디자인 참고)
- 매일 자동: 캐시 증분 갱신 → 스크리너(index + 종목 상세) 재생성 → Pages 배포.
- 백테스트는 무거워 주간/수동.
- 헤더에 **데이터 기준일**(마지막 거래일) 표기 권장.

## 7. 디자인에 바라는 것 (자유도)
- 위 정보 구조는 유지하되, **시각적 위계·여백·타이포·반응형**은 더 세련되게 제안 환영.
- 특히 **스크리너 표의 가독성**(수백 행을 빠르게 훑기)과 **상세의 차트+카드 균형**이 핵심.
- 다크모드 토글(선택). 인쇄/스크린샷 친화(상세 1장 캡처).
