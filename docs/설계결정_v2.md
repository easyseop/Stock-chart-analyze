# 설계 결정 (v2) — 구현 확정 사항

> 원본 명세(`설계명세_원본.md`)를 구현 가능한 수준으로 구체화한 결정 기록.
> 리뷰에서 드러난 구멍을 메우고 수치를 확정했다.

## 1. 점수 → 판정 : 100점 정규화
- 점수 기여 모듈 4개: `trend(MA·다중TF) / rsi / sr(지지저항·박스) / volume(거래대금)`
- **ADX = 국면 게이트**(점수 X), **ATR = 손절 계산용**(점수 X)
- 정규화: `100 × Σ(모듈점수×가중치) / Σ(2×가중치)` → −100 ~ +100

| 정규화 점수 | 판정 | 게이지 |
|---|---|---|
| ≥ +50 | 적극 매수 | 🟢 강세 |
| +20 ~ +50 | 매수 관심 | 🟢 |
| −20 ~ +20 | 관망 | ⚪ 중립 |
| −50 ~ −20 | 매도 관심 | 🔴 |
| ≤ −50 | 적극 매도/회피 | 🔴 공포 |

경계는 위쪽 포함(`lo ≤ x < hi`)으로 중복 제거.

## 2. 국면별 가중치 매트릭스
| 모듈 | 추세장(ADX>25) | 횡보장(<20) | 전환(20~25) |
|---|---|---|---|
| trend | ×1.0 | ×0.5 | ×0.5 |
| rsi(역추세) | ×0.0 | ×1.0 | ×0.5 |
| sr(지지저항) | ×0.5 | ×1.0 | ×0.5 |
| volume(돌파) | ×1.0 | ×0.5 | ×0.5 |

→ 추세장에선 RSI 과매수/과매도를 **점수 0으로 무시**(추세 지속 존중).

## 3. 시간프레임 : 월·주·일 3개 (년봉 제외)
년봉은 MA 계산에 데이터가 절대 부족 → 점수에서 제외. 봉별 MA 기간 차등:
- 일봉 `5/20/60/120`, 주봉 `5/10/20/60`, 월봉 `3/6/12/24`
- 다중TF 정렬: 각 TF 대표선(일20·주20·월12) 기울기+위치로 방향 → 셋 다 ↑ +2 / 엇갈림 0 / 셋 다 ↓ −2

## 4. 박스권 + 핵심 방어선(이탈금지선) — 사용자 요청 추가
- 박스 탐지: 후보 구간(20/60일) 중 좁고(`(고−저)/저 < 15%`) 경계 터치 많은 구간 선택
- **경계는 직전까지(현재 봉 제외)로 산출** → 현 시점의 이탈을 잡기 위함
- 방어선 = 박스 하단. POC·MA60/120·라운드넘버와 겹치면 신뢰도 ↑(강/보통/약)
- 점수: 바닥권 반등 +2 / 상단 돌파 +2 / 상단 임박 −1 / 중단 0 / **하단 이탈 −2**
- 카드에 **박스권·핵심방어선 항상 노출**

## 5. 거래대금 (위치 × 캔들)
- 거래대금 = 종가×거래량, `당일 > 20일 평균 ×2` 급증
- 위치(sr 모듈 재사용)×캔들로 의미 확정: 돌파+양봉 +2 / 바닥권 폭발 양봉 +1(셀링클라이맥스) / 고점권 장대음봉 −2 / 거래없는 돌파 0(가짜 의심)

## 6. 리스크 / 포지션
- 진입가: 고점권(저항 임박)=박스 상단(돌파 시 매수), 그 외=현재 종가
- 손절: `진입 − 2·ATR` 와 `방어선 ×0.99`(이탈 확인 버퍼) 중 STOP_MODE에 따라 선택(기본 nearest)
- 1차 목표: `진입 + 2×손절폭` (손익비 1:2 정확)
- 비중: `계좌 × 1% ÷ 손절폭` (통화별 가상 계좌)
